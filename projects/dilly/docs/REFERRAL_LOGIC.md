# Invite a Friend — Referral Logic

## Overview

**Reward:** Give a friend 1 free month, you get 1 free month.

- Each user has a unique `referral_code` (8 chars, URL-safe) stored in their profile.
- Shareable link: `/?ref=CODE` or `/invite/CODE` (redirects to `/?ref=CODE`).
- When a referred user subscribes, both get 1 free month.

## Implementation Status

### Done
- `referral_code` generated per user (in `profile_store.py`, `ensure_profile_exists` / `ensure_referral_code`)
- PATCH `/profile` accepts `referral_code` (read-only from client; backend generates)
- Settings: "Invite a friend" section with copy link button
- `/invite/[code]` route redirects to `/?ref=code`

### Pending (Stripe integration)
- **Attribution:** On sign-up or first subscription, store `ref` param (e.g. in session or profile as `referred_by`).
- **Reward application:** When referred user completes first payment:
  - Extend referred user's subscription by 1 month (or grant 1 free month).
  - Extend referrer's subscription by 1 month (or grant 1 free month).
- **Webhook:** Stripe `customer.subscription.created` or `invoice.paid` — look up `referred_by`, apply rewards, clear `referred_by` to prevent double-credit.

### Logic (when Stripe is live)

1. User A shares link `meridian.com/?ref=ABC123`.
2. User B clicks, signs up, subscribes. During checkout or profile creation, store `referred_by: "ABC123"` (or the referrer's profile id).
3. On User B's first successful payment:
   - Grant User B 1 free month (e.g. extend subscription end date by 30 days).
   - Look up User A by `referral_code === "ABC123"`, grant User A 1 free month.
   - Clear `referred_by` on User B's profile (or mark as "rewarded").
4. Track in profile or separate table: `referral_rewards_applied: true` to avoid double-crediting.

## Profile Fields

- `referral_code`: string, 8 chars, generated by backend.
- `referred_by`: string (referrer's code), set when user signs up via ref link. Used at first payment to apply rewards.
