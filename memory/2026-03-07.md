### Session 3 — Batch 2 (Voice overhaul, profile edit, clickable history, collapsible report, condensed onboarding)

**Voice Quality Overhaul** — The biggest change:
- Completely rewrote `_VOICE_SYSTEM` prompt in `api/main.py` — no longer generic chatbot advice. Now demands evidence-based, resume-specific answers with concrete project/certification recs, bullet rewording (Before→After), score targets, and peer context.
- Frontend now sends full audit context to Voice: audit_findings, recommendations (including line_edits), peer_percentiles, final_score, application_target, name — not just bare scores.
- Backend parses and formats all this rich context for the LLM: "THIS USER'S RESUME AUDIT DATA" section with findings, line edits, strategic recs.
- Increased max_tokens from 800→1200, temperature from 0.4→0.35 for more consistent, substantive answers.
- Voice suggestion chips are now dynamic based on user's actual data (e.g. "My Grit is 42 — how do I raise it?" instead of generic "How can I improve my Grit score?")

**Profile Editing** — Career Center header now has a pencil icon that opens inline edit form for name and major. Uses existing `PATCH /profile` endpoint.

**Clickable History** — History items in Career Center are now buttons that fetch the full audit by ID and open the report:
- Backend: `append_audit` now stores full audit response (not just summary)
- Backend: New `GET /audit/history/{audit_id}` endpoint returns full audit by ID
- Frontend: History items are clickable with hover state and chevron indicator

**Collapsible Report Sections** — Assessment Findings, Strategic Recommendations, and Evidence sections use native `<details open>` elements. All start expanded, user can collapse to focus. Added CSS in `globals.css` for styled chevron indicators.

**Condensed Onboarding** — Steps 9 (motivational) and 10/11 (resume upload) merged into one screen. Personalized bullets now appear as compact pills above the upload zone. One fewer tap to complete onboarding.
