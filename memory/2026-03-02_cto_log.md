# CTO Log - 2026-03-02
## Project: Dilly SaaS Wrapper - Peer-Relative Scoring Update

### 1. Architectural Shift: Relative vs. Absolute
I have pivoted the scoring logic from "Absolute Ideal" (comparing candidates to a high-achieving hypothetical baseline) to "Peer-Relative" (comparing candidates against each other within the same track).

### 2. Implementation Details
- **Percentile-Based Engine:** Added `get_cohort_percentile` to `api/main.py`. This function now queries the `beta_cohort_db.json` (100+ candidates) in real-time.
- **Track-Specific Pool:** When a candidate is audited (e.g., as a "Builder"), their scores are now calculated as a percentile of only the other "Builders" in the database.
- **Dynamic Evidence:** The `evidence` field now explicitly states the candidate's rank (e.g., *"Performance exceeds 86% of peers in leadership..."*) rather than just listing raw markers.

### 3. Impact on MTS (Dilly Truth Standard)
This maintains the MTS while making the data more actionable for recruiters and students. A score of "90" now truly means "Top 10% of the cohort," providing a realistic competitive landscape rather than an unreachable standard.

**Update (2025-03-04):** Beta cohort removed. Dilly now uses only LLM few-shot examples (`prompts/training_data.json`). No cohort DB; `/audit` returns raw scores; `/audit/v2` is the primary endpoint.
